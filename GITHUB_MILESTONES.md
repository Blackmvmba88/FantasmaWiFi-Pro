## Issue Labels

### Priority
- `priority: critical` - Must be done immediately
- `priority: high` - Important for milestone
- `priority: medium` - Should be done if time permits
- `priority: low` - Nice to have

### Type
- `type: feature` - New feature
- `type: enhancement` - Improvement to existing feature
- `type: bug` - Bug fix
- `type: documentation` - Documentation update
- `type: infrastructure` - Build, CI/CD, tooling

### Area
- `area: cli` - Command-line interface
- `area: web-ui` - Web user interface
- `area: api` - REST API
- `area: core` - Core functionality
- `area: plugins` - Plugin system
- `area: adapters` - Platform adapters
- `area: docs` - Documentation
- `area: tests` - Testing
- `area: install` - Installation and packaging

### Quarter
- `Q1-2026` - Q1 2026 milestone
- `Q2-2026` - Q2 2026 milestone
- `Q3-2026` - Q3 2026 milestone
- `Q4-2026` - Q4 2026 milestone
- `2027` - Long-term vision

### Status
- `status: planned` - Planned, not started
- `status: in-progress` - Currently being worked on
- `status: blocked` - Blocked by dependency
- `status: review` - In review
- `status: completed` - Completed

### Special
- `good first issue` - Good for newcomers
- `help wanted` - Need community help
- `roadmap` - Roadmap item
- `breaking change` - Breaking API change
- `security` - Security-related
